# sim-db
Rudimentary database for config files for wfpsim and gcsim in plaintext. 

Namely, this would be for the websites https://wfpsim.com and https://gcsim.app, albeit many of these will work for forks of those as well.

Created mostly because wfpsim doesn't have one, and their links expire in a month. Also because this'll allow me to more easily programatically sweep through all of my sims with KQMS/stat/format checks, mass run the substat optimizer for KQMS ones (AND keep the freaking human-readable number format intact!), analyze for both granular and mass data, allow for a versioning system, and hopefully - HOPEFULLY - allow for easier collaboration.

The fact that there's a versioning system here should also hopefully serve as incentive for people to submit their gcsim configs.

Largely Noelle-focused at the moment due to the creator's bias, but meant for any character and any non-trivial sim, including very low DPS ones, "whalecrafting", non-KQMS, ones proving a certain team has little synergy, ones trying to optimize purposefully silly teams... and so on.

## Usage notes ##
1. The idea is for this to be open for contribution by anyone, both for applying new sims as well as improving current ones. However, new ones will be verified for at least *logical sense* before being added, and improvements will be checked for. Please also do specify who worked on it, as I'd rather not have sims misattributed to me... albeit that worry may be unnecessary given the sheer amount of characters I'll be slowly including. Note that sims **may be largely unoptimized, be in rotation or stats**, or even submitted as WIPs.

2. Files .simpart are all plaintext and can be opened with any text editor - in fact, I encourage the usage of characters-copypaste.simpart for easier wfpsim writing. (Or you could use GitHub's search function in this DB! That requires having an account and logging in, though.) 

3. When submitting, imperfection is not a problem, and is, in fact, expected. Iterating on and improving sims is a natural part of the process, and having a github DB makes updating files (or creating ones) trivial, while also keeping a history of past versions in case you want to go back. As long as a sim at least follows a basic skeleton and makes sense, I don't mind including it.

If you just want your sim saved somewhere online/easily share it but don't want to go through the trouble of using GitHub, feel free to just DM me at @pkach404 on Discord with the wfp link or config file.
